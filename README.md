# TransitOps 🚛
An enterprise-grade, full-stack Decoupled Fleet & Transit Management system featuring a premium React/Tailwind CSS frontend driven by a robust Node.js, Express, and MongoDB backend monolith.
### 🔗 Live Demo: [TransitOps_login](https://transitops-final.onrender.com/login)

---

## 🏗️ Architecture Overview
TransitOps is engineered using a clean, decoupled full-stack architecture built to optimize performance and strictly enforce enterprise business boundaries:
* **Frontend UI Desk:** A highly responsive Single Page Application (SPA) powered by React, Vite, and Tailwind CSS, utilizing Zustand for global reactive state synchronization and Axios api routing.
* **Backend API Monolith:** An Express app framework utilizing Mongoose schemas to interface with a MongoDB instance, strictly managing state validations and secure business transitions.
* **Seed Isolation:** Includes an automated data injection pipeline (`seeder.js`) that safely provisions base assets and 12 distinct functional enterprise user accounts upon system initialization.

---

## 👥 Targeted Enterprise Roles & Access Control
The application securely structures logistics access through custom tokens generated by the JSON Web Token (JWT) workflow, providing restricted views for four targeted operations:
1. **Fleet Manager:** Full administrative CRUD access over all vehicle registries, active field configurations, and logistics desks.
2. **Driver:** Context-restricted operational views allowing personnel to review active dispatch metrics and update live trip status logs.
3. **Safety Officer:** Dedicated analytical overview of operator records, handling license compliance states and safety performance values.
4. **Financial Analyst:** Read-only visibility over overhead logs, auditing fuel ledgers, toll expenses, and capital asset ROI margins.

---

## 🚀 Key Features & Validation Constraints
* **Interactive Mission Control Pipeline:** A multi-step visual tracking board (`Draft` -> `Dispatched` -> `Completed` -> `Cancelled`) to monitor active transport lines in real-time.
* **Automated Workflow Rules:** Server-side schema controllers that explicitly validate cargo configurations to prevent overloading, verify driver license compliance dates, and lock vehicles under active maintenance into an `In Shop` flag state.
* **Hardened Security Boundaries:** Built-in network defense triggers embedded inside the authentication router that enforce a strict 5-attempt login rate limit to neutralize brute-force activities.
* **Data-Oriented Telemetry Engine:** Advanced SVG data visualization graphs updating dynamically from database operations, accompanied by a serverless exporting module producing spreadsheet-ready `.csv` reports on demand.
